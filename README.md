## рҹҺЁ Personalizar la marca (branding)

El panel acepta **un archivo `branding.json`** con tu propia marca: nombre, logo, colores, tagline, copyright, etc. Ideal si revendes o configuras el panel para un cliente especГӯfico.

Copia `branding.example.json` a `branding.json` y edГӯtalo:

```json
{
  "brand": {
    "name": "TU EMPRESA DE SOPORTE",
    "tagline": "Soporte tГ©cnico para pymes",
    "companyUrl": "https://miempresa.com",
    "copyright": "В© 2026 Mi Empresa"
  },
  "theme": {
    "accentColor": "#0078D4",
    "darkMode": false,
    "cornerRadius": 4
  },
  "window": {
    "title": "Panel Mi Empresa v1.0"
  }
}
```

Rutas donde el launcher busca `branding.json` (en orden):

1. `branding.json` junto al `launcher.ps1` (repo local).
2. `%LOCALAPPDATA%\AtlasPC\branding.json` (perfil del usuario).
3. `%APPDATA%\AtlasPC\branding.json`.

Si no existe, se usa el branding Atlas por defecto.

рҹ“ҳ Ver detalles en [docs/BRANDING.md](docs/BRANDING.md).

### Idiomas

### AГұadir una nueva herramienta

1. Crea `src/tools/Invoke-MiTool.ps1`:
   ```powershell
   function Invoke-MiTool {
       [CmdletBinding()] param()
       Write-Host "Hola desde MiTool"
       Read-Host "Presiona Enter para salir"
   }
   ```
2. AГұade la entrada en `config/tools.json`:
   ```json
   {
     "id": "mi-tool",
     "name": "Mi Tool",
     "description": "DescripciГіn corta (mГЎx. ~150 caracteres).",
     "category": "mantenimiento",
     "function": "Invoke-MiTool",
     "source": "MiTool.ps1",
     "requiresAdmin": false,
     "runsInNewWindow": true,
     "dependencies": []
   }
   ```
3. Ejecuta `pwsh -File build.ps1`.
4. Commit y push вҖ” CI regenera `launcher.ps1` automГЎticamente.

Ver [docs/ADDING-TOOLS.md](docs/ADDING-TOOLS.md) para detalles.
